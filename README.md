# 江苏省办销售 Pipeline 本地分析工作台

纯前端本地看板。用户在浏览器上传 Excel 后，数据只在本机浏览器内解析和分析，不依赖后端或数据库。

## 运行

```bash
npm install
npm run dev
```

访问：http://127.0.0.1:5173/

生产构建：

```bash
npm run build
```

## 已实现的 MVP 能力

- 上传 `.xlsx / .xls / .csv`，限制 20MB。
- 支持拖拽 Excel 到页面直接导入统计。
- 识别 `PPL明细`、`数据汇总`、`新增PPL+活动记录`。
- PPL 字段别名映射、金额/赢单率标准化、导入报告。
- 6 个 KPI：商机总数、商机总金额、客户数、加权赢单率、Forecast 金额、风险商机数。
- 6 个核心图表：销售 TOP 10、行业分布、产品 TOP 15、季度分布、健康度金额与商机数分布、Forecast 占比。
- 手动筛选、指定客户筛选、全局搜索、图表点击下钻、下钻标签移除。
- PPL 明细支持按客户、行业、产品、销售阶段聚合统计，也可切回明细数据。
- 明细表分页、排序、行详情抽屉、当前筛选结果 CSV 导出。
- 重点客户分析 Tab：支持输入多个客户、客户名称标准化匹配、别名匹配、模糊匹配、匹配结果展示、未匹配提示、客户 KPI、客户图表和重点客户明细导出。

## 重点客户分析

在 `重点客户分析` Tab 中输入客户名单，每行一个客户，也可以使用逗号、顿号或分号分隔。系统会基于 PPL 明细中的客户名称进行：

- 标准化后的精确匹配。
- 内置客户别名匹配。
- 包含式模糊匹配。

客户名称标准化会去除空格、括号、常见公司后缀，如 `有限公司`、`股份有限公司`、`有限责任公司`、`集团`、`总部` 等。

重点客户分析独立于 PPL 明细 Tab，不会影响已有全局筛选、图表下钻和聚合统计。

## 已知限制

- 当前 Excel 解析使用 `xlsx@0.18.5`，npm audit 报告该包存在高危漏洞且无官方修复版本。MVP 已做文件大小、Sheet/行列数量、危险 key 过滤和不渲染 HTML 等限制；生产交付版建议替换解析库或将解析隔离到 Web Worker。
- 当前实现是可演示 MVP，Excel 解析仍在主线程；大文件生产使用应补 Web Worker。
- 数据汇总和活动记录视图当前以表格预览为主，后续可继续补完整管理图表。
- 当前活动记录 Sheet 没有客户名称字段，因此重点客户分析中的活动记录数暂显示为无客户维度，不按销售分摊，避免误导。
